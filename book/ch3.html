РОЗДІЛ 3. Виведення даних

3.1 Навіщо програмі виводити дані?

Програма може виконувати мільйони обчислень за секунду, але якщо вона не покаже результат, ми про нього ніколи не дізнаємося. Виведення інформації — це процес передачі даних від комп'ютера до користувача (через екран, колонки, принтер тощо).

Усі ваші улюблені програми постійно щось виводять:

  • месенджери виводять текст повідомлень вашого друга;
  • ігри виводять графіку, рахунок та кількість «життів» персонажа;
  • калькулятор виводить результат обчислень.

Коли ми тільки починаємо вчити Python, вікном для спілкування з комп'ютером стає консоль (термінал, вікно результатів виконання програми) — просте текстове віконце, куди програма виводить свої «думки» у вигляді літер і чисел.

Порівняйте дві програми:

2 + 2  # Комп'ютер порахував, але на екран нічого не вивів

print("Привіт! Я твій комп'ютер.")  # Одразу бачимо текст на екрані

У першому випадку обчислення відбулося, але результат ніде не з'явився — комп'ютер «мовчить». У другому — ми явно попросили вивести текст, і він з'явився в консолі.

3.2 Функція print()

У Розділі 1 «Вступ» ми познайомилися з функцією print(), яка виводить рядок (відомий «Привіт, світе!»). Слово print англійською означає «друкувати». Дужки () після слова print вказують на те, що це функція (команда), а лапки всередині дужок кажуть комп'ютеру: «просто візьми цей текст і перенеси його на екран один в один».

Насправді, функція print() виводить аргумент, переданий у дужках, та символ нового рядка. Цей перехід на новий рядок додається за замовчуванням. Якщо ж ви не хочете виводити цей символ, можна використати іменований аргумент end (докладніше — у §3.2.4):

print("Привіт, світе!")
print("Привіт, світе!", end="")

🖥 Результат:

Привіт, світе!
Привіт, світе!

3.2.1 Текстові повідомлення та лапки

У програмуванні будь-який текст (набір символів) називається рядком (англійською string). Щоб Python зрозумів, де текст починається і де закінчується, його обов'язково беруть у лапки. Якщо лапки пропустити, Python подумає, що це назва команди, і заплутається.

У Python можна використовувати як подвійні, так і одинарні лапки — для комп'ютера вони абсолютно однакові:

print("Python")
print('Python')

Обидва рядки виведуть слово Python. Головне правило: якими лапками рядок відкрили, такими й закриваємо. Наприклад, код print("Привіт') не працюватиме, бо лапки різні.

3.2.2 Виведення чисел та обчислень

Комп'ютер «обожнює» числа, тому для них лапки не потрібні: якщо поставити число в лапки, Python сприйме його просто як текст (рядок), а не як число. У Python числа поділяються на два основні типи (детальніше в Розділі 2): цілі числа (int) — без дробової частини, та дійсні (дробові) числа (float) — з крапкою.

У Python для розділення цілої та дробової частини використовується крапка, а не кома. 
print(25)    # Ціле число
print(3.14)  # Дійсне число (число Пі)

Python — це ще й потужний калькулятор. Якщо всередині print() написати математичний вираз без лапок, комп'ютер спочатку виконає обчислення, а вже потім виведе кінцевий результат:

print(7 + 5)        # Виведе: 12
print(10 * 4)       # Виведе: 40 (зірочка * — це множення)
print((8 + 2) / 5)  # Виведе: 2.0 (коса риска / — це ділення)

Дужки керують пріоритетом дій так само, як і на уроках математики.

3.2.3 Виведення значень змінних

Функція print() також може виводити вміст будь-якої змінної, незалежно від її типу. Уявіть, що змінна — це коробка з назвою, у якій зберігається якась інформація; ми можемо покласти туди число чи текст, а потім попросити print() показати, що всередині:

age = 13    # Створили "коробку" з назвою age і поклали туди 13
print(age)  # Виведе те, що лежить у коробці, тобто: 13

Для цілого числа так само:

zminna = 3
print(zminna)

🖥 Результат:

3

Порівняйте два рядки:

print(age)    # Виведе: 13 (вміст змінної)
print("age")  # Виведе: age (просто текст у лапках)

Лапки мають «магічне» значення: є лапки — друкуємо текст як є; немає лапок — Python шукає змінну з таким ім'ям. Якщо такої змінної не існує, ми отримаємо помилку NameError .

Можна вивести вміст кількох змінних одразу (незалежно від їхнього типу), розділивши їх комами:

x = 32
nazva = "Яблука"
print(nazva, "вартістю", x, "грн/кг")

🖥 Результат:

Яблука вартістю 32 грн/кг

Python «написав» повне речення, замінивши змінні x та nazva їхнім вмістом. Зверніть увагу: для виведення кількох елементів в одному рядку ми використали роздільник , між ними, і Python автоматично додав пробіл на місці кожної коми. Цю поведінку можна змінити за допомогою іменованого аргументу sep .

3.2.4 Параметр sep — роздільник

Коли ми виводимо кілька значень через кому, Python за замовчуванням розділяє їх пробілом. Щоб змінити цю поведінку — наприклад, поставити дефіс, вертикальну риску чи взагалі прибрати роздільник — використовують іменований аргумент sep (від англійського separator, роздільник), який записується наприкінці через кому:

x = 32
nazva = "Яблука"
print(nazva, "вартістю", x, "грн/кг", sep="")
print(nazva, "вартістю", x, "грн/кг", sep="-")
print(nazva, "вартістю", x, "грн/кг", sep="_")

🖥 Результат:

Яблукавартістю32грн/кг
Яблука-вартістю-32-грн/кг
Яблука_вартістю_32_грн/кг

Ще кілька прикладів:

print(1, 2, 3)            # Без sep (автоматичний пробіл): 1 2 3
print(1, 2, 3, sep="-")   # Роздільник дефіс: 1-2-3
print(1, 2, 3, sep=" | ") # Роздільник вертикальна риска: 1 | 2 | 3
print(1, 2, 3, sep="")    # Без роздільника взагалі: 123

Практичні приклади використання sep:

  • дата: print(25, 10, 2026, sep=".") → 25.10.2026;
  • координати: print("X", 45, "Y", 12, sep=":") → X:45:Y:12;
  • список чисел: print("Номери", 1, 2, 3, sep="#") → Номери#1#2#3.

Щоб вивести два рядки один біля одного без пробілу, можна або конкатенувати їх, або використати sep з порожнім рядком:

tvaryna1 = "кіт"
tvaryna2 = "миша"
print(tvaryna1, tvaryna2)
print(tvaryna1 + tvaryna2)
print(tvaryna1, tvaryna2, sep="")

🖥 Результат:

кіт миша
кітмиша
кітмиша

3.2.5 Параметр end

Зазвичай після виконання print() курсор автоматично переходить на новий рядок, і наступний print() починає писати вже знизу:

print("Привіт")
print("Python")
# Результат у консолі:
# Привіт
# Python

Якщо ми хочемо заборонити Python переходити на новий рядок, використовуємо іменований аргумент end (від англійського end, кінець), який вказує, чим завершити виведення (за замовчуванням — символом нового рядка):

print("Привіт", end=" ")
print("Python")
# Результат у консолі:
# Привіт Python

Інший спосіб побачити ефект end — використати дві функції print() в одному рядку через ;, який слугує роздільником інструкцій Python в одному рядку:

print("Привіт"); print("Інна")
print("Привіт", end=""); print("Інна")
print("Привіт", end=" "); print("Інна")

🖥 Результат:

Привіт
Інна
ПривітІнна
Привіт Інна

Приклад використання end із циклом (докладніше про цикли):

for i in range(5):
    print(i, end=" ")
# Виведе все в один рядок: 0 1 2 3 4

3.2.6 Керувальні символи

Іноді потрібно перенести текст на новий рядок або зробити відступ (табуляцію) всередині одного рядка в лапках. Для цього використовують спеціальні поєднання символів, що починаються зі зворотної похилої риски \:

  • \n — перехід на новий рядок (newline);
  • \t — табуляція (кілька пробілів для вирівнювання тексту);
  • \" — дозволяє надрукувати подвійну лапку всередині рядка, обмеженого подвійними лапками;
  • \\ — дозволяє надрукувати саму зворотну похилу риску.

Приклади:

print("Перший рядок\nДругий рядок")
# Виведе:
# Перший рядок
# Другий рядок

print("Ім'я\tВік")
print("Марко\t14")
# Виведе охайну табличку:
# Ім'я    Вік
# Марко   14

print("Він сказав: \"Привіт!\"")  # Виведе: Він сказав: "Привіт!"

3.2.7 Символи Unicode та емоційки

Python — сучасна мова, яка чудово розуміє міжнародний стандарт символів Unicode. Це означає, що можна виводити в консоль не лише звичайні літери, а й емоційки, прапори країн та інші спеціальні знаки:

print("😀")
print("🇺🇦")
print("Програмування — це цікаво! 🚀")

Такі символи можна просто копіювати й вставляти всередину лапок у коді.

3.2.8 Оформлення виведення

Коли програма показує результати, вони мають бути зручними для читання. Охайне оформлення — це прояв поваги до користувача. За допомогою дефісів, знаків рівності, зірочок та тексту можна створювати рамки, заголовки й меню:

name = "Олена"
print("Ім'я:", name)
print("----------------")
print("=== Меню ===")
print("1. Старт гри")
print("2. Налаштування")
print("3. Вихід")
print("----------------")

3.3 Повідомлення про помилки

Ми вже стикалися з повідомленнями про помилки в попередньому розділі про змінні. Тут ми пояснимо, як їх читати. Помилятися — це нормально: навіть професійні програмісти щодня роблять помилки. Головне — навчитися читати повідомлення від Python (вони називаються Error), щоб швидко їх виправляти.

Починаючи з версії 3.10 Python, інтерпретатор повертає зрозумілі повідомлення при синтаксичних помилках. Розглянемо скрипт (збережений у файлі test.py), який містить кілька помилок:

 print("кіт"
 print("миша")
 print(1 / 0)
 print(int("два"))

Ви, мабуть, помітили відсутність закриваючої дужки в рядку 1. Коли ми запускаємо скрипт, ми отримуємо:

$ python test.py
  File "test.py", line 1
    print("кіт"
            ^
SyntaxError: '(' was never closed

Як читати повідомлення про помилку (Traceback)? Це завжди робиться знизу вгору:

  • У самому низу: тип згенерованої помилки (пізніше ми побачимо, що це називається винятком, або exception). Тут — синтаксична помилка SyntaxError, а в тому самому рядку — додаткова підказка (відсутність закритої дужки).
  • Трохи вище: опис помилки, де показано відкриту дужку, яка так і не була закрита.
  • Ще вище: номер рядка в коді, де було виявлено помилку.

З таким Traceback стає легко виправити помилку.

Якщо ми виправимо рядок 1, додавши кінцеву дужку, і знову запустимо скрипт, отримаємо помилку через ділення на нуль:

$ python test.py
кіт
миша
Traceback (most recent call last):
  File "test.py", line 3, in <module>
    print(1 / 0)
          ~~^~~
ZeroDivisionError: division by zero

Якщо виправити й цю помилку (наприклад, print(1 / 1)), виконання видасть інше повідомлення через рядок 4, де перетворення рядка на ціле число неможливе:

$ python test.py
кіт
миша
1
Traceback (most recent call last):
  File "test.py", line 4, in <module>
    print(int("два"))
          ^^^^^^^^^
ValueError: invalid literal for int() with base 10: 'два'

У всіх наведених прикладах структура Traceback однакова: у самому низу — тип помилки, потім, піднімаючись вгору, — опис проблеми та номер рядка, де її виявлено.

Важливо уважно читати кожне повідомлення про помилку, згенероване Python. Зазвичай ключ до проблеми міститься саме в цьому повідомленні.

3.3.1 Типові помилки початківців

Помилка 1. Забули або не закрили лапки

print("Привіт)
# Помилка: SyntaxError: unterminated string literal

Як виправити: перевірити, щоб наприкінці рядка стояла та сама лапка, що й на початку: print("Привіт").

Помилка 2. Пропустили дужку

print("Світ"
# Помилка: SyntaxError: '(' was never closed

Як виправити: додати закриваючу дужку ). Скільки дужок відкрили — стільки маємо й закрити.

Помилка 3. Переплутали ім'я змінної з текстом

my_car = "Tesla"
print(my_car)  # Правильно — виведе Tesla
print(my_сar)  # Помилка: NameError: name '...' is not defined

Як виправити: уважно перевірити кожну літеру в назві змінної. Помилка NameError майже завжди означає друкарську помилку в імені змінної або використання літери з іншої розкладки клавіатури (наприклад, українська «с» замість англійської «c»).

3.4 Форматоване виведення та f-рядки

3.4.1 Визначення

Визначення. Форматоване виведення — це механізм, який дозволяє виводити змінні з точним форматуванням: наприклад, вирівняними ліворуч або праворуч, або з певною кількістю десяткових знаків для float. Форматоване виведення незамінне, коли потрібно створювати файли чи звіти, організовані у «красиві стовпчики» — таблиці з результатами, списки товарів з цінами тощо.

Починаючи з версії 3.6, Python запровадив f-рядки для реалізації форматованого виведення. Існують і старіші способи форматування рядків (розглянуті в Розділі 26 «Додаткові зауваження», онлайн), однак для початківців рекомендовано одразу використовувати f-рядки — вивчати старі методи немає сенсу.

Визначення. f-рядок — скорочення від formatted string literals. У попередньому розділі ми бачили звичайні рядки (strings) — текст, укладений в одинарні або подвійні лапки, наприклад "Це рядок". Еквівалент у вигляді f-рядка — той самий рядок, перед яким без пробілу стоїть символ f:

"Це рядок"
f"Це рядок"

Символ f перед лапками вказує Python, що це f-рядок, який задіює механізм форматованого виведення, на відміну від звичайного рядка. Детальніше про механізм, що лежить в основі — у Розділі 11 «Більше про рядки».

3.4.2 Використання f-рядків

f-рядки дозволяють краще організувати виведення змінних. Повернемося до прикладу з Джоном:

x = 32
nazva = "Яблука"
print(f"{nazva} вартістю {x} грн/кг")

🖥 Результат:

Яблука вартістю 32 грн/кг

Достатньо передати ім'я змінної всередині пари фігурних дужок, і Python замінить їх вмістом. Синтаксис виглядає читабельнішим, ніж еквівалент через кому:

x = 32
nazva = "Яблука"
print(nazva, "вартістю", x, "грн/кг")

🖥 Результат:

Яблука вартістю 32 грн/кг

Не забувайте f перед першими лапками, інакше Python сприйме рядок як звичайний і не запустить механізм заміни у фігурних дужках:

x = 32
nazva = "Яблука"
print("{nazva} вартістю {x} грн/кг")

🖥 Результат:

{nazva} вартістю {x} грн/кг

Змінну можна використовувати більше одного разу в одному f-рядку:

zminna = "то"
print(f"{zminna} та {zminna} утворюють {zminna}{zminna}")

🖥 Результат:

то та то утворюють тото

У фігурні дужки можна поміщати й числові значення або рядки безпосередньо:

print(f"Я виводжу ціле {10} та float {3.14}")
print(f"Я виводжу рядок {'Python'}")

🖥 Результат:

Я виводжу ціле 10 та float 3.14
Я виводжу рядок Python

Це працює з будь-яким типом змінних. Зверніть увагу: для рядків усередині фігурних дужок використовуйте одинарні лапки, якщо сам f-рядок визначено подвійними.

3.4.3 Специфікація формату

f-рядки дозволяють не тільки замінювати змінні всередині рядка, а й вказувати формат їхнього виведення. Наприклад, обчислимо й виведемо частку правильних відповідей у тесті (4500 балів за правильні відповіді, 2575 — за частково правильні, максимум — 14800):

chastka_pravylnykh = (4500 + 2575) / 14800
print("Частка правильних відповідей становить", chastka_pravylnykh)

🖥 Результат:

Частка правильних відповідей становить 0.4780405405405405

Результат має занадто багато десяткових знаків. Щоб записати його читабельніше, вкажіть у фігурних дужках {} бажаний формат:

chastka_pravylnykh = (4500 + 2575) / 14800
print(f"Частка правильних відповідей становить {chastka_pravylnykh:.2f}")
print(f"Частка правильних відповідей становить {chastka_pravylnykh:.3f}")

🖥 Результат:

Частка правильних відповідей становить 0.48
Частка правильних відповідей становить 0.478

Розберемо вміст фігурних дужок {chastka_pravylnykh:.2f}:

  • спочатку йде ім'я змінної для форматування — chastka_pravylnykh (обов'язкове для f-рядків);
  • далі двокрапка :, яка вказує, що далі буде вказано формат виведення;
  • праворуч від двокрапки — .2f, де f означає, що змінну виводимо як float, а .2 — бажану точність (дві цифри після коми).

Форматування .xf (де x — додатне ціле число) повертає округлений результат. Цілі числа форматують літерою d (decimal integer):

kilk_pravylnykh = 4500
print(f"У тесті {kilk_pravylnykh:d} правильних відповідей")

🖥 Результат:

У тесті 4500 правильних відповідей

Можна помістити й кілька чисел в один рядок; якщо інструкція довга, її можна розбити на кілька рядків коду, щоразу ставлячи f перед кожним фрагментом рядка:

chastka_pravylnykh = (4500 + 2575) / 14800
kilk_pravylnykh = 4500
kilk_chastkovykh = 2575
print(f"У тесті {kilk_pravylnykh:d} правильних та {kilk_chastkovykh:d} часткових відповідей,"
      f"що становить частку {chastka_pravylnykh:.2f}")
vidsotok_pravylnykh = chastka_pravylnykh * 100
print(f"У тесті {kilk_pravylnykh:d} правильних та {kilk_chastkovykh:d} часткових відповідей,"
      f"тобто {vidsotok_pravylnykh:.2f}% успішності")

🖥 Результат:

У тесті 4500 правильних та 2575 часткових відповідей,що становить частку 0.48
У тесті 4500 правильних та 2575 часткових відповідей,тобто 47.80% успішності

Зверніть увагу: оскільки текст просто «склеюється» з двох сусідніх f-рядків, між комою та наступним словом немає пробілу, якщо його явно не додати на початку другого фрагмента (наприклад, f" що становить...").

Нарешті, можна вказати, на скільки символів має бути записаний результат і як відбувається вирівнювання (ліворуч, праворуч, по центру):

print(10); print(1000)
print(f"{10:>6d}"); print(f"{1000:>6d}")
print(f"{10:<6d}"); print(f"{1000:<6d}")
print(f"{10:^6d}"); print(f"{1000:^6d}")
print(f"{10:*^6d}"); print(f"{1000:*^6d}")
print(f"{10:0>6d}"); print(f"{1000:0>6d}")

🖥 Результат:

10
1000
    10
  1000
10    
1000  
  10  
 1000 
**10**
*1000*
000010
001000

> вказує на вирівнювання праворуч, < — ліворуч, ^ — по центру. Можна також вказати символ-заповнювач (за замовчуванням — пробіл).

Це форматування можливе й для рядків, з літерою s (string):

print("товар: ручка"); print("товар: зошит")
print(f"товар: {'ручка':>7s}"); print(f"товар: {'зошит':>7s}")

🖥 Результат:

товар: ручка
товар: зошит
товар:   ручка
товар:   зошит

Форматоване виведення дозволяє писати ідеально вирівняні стовпчики — дуже зручно, наприклад, для списку товарів з цінами у вигляді таблиці.

Для float можна комбінувати загальну кількість символів з кількістю десяткових знаків:

vidsotok_pravylnykh = ((4500 + 2575) / 14800) * 100
print(f"{vidsotok_pravylnykh:7.3f}")
print(f"{vidsotok_pravylnykh:10.3f}")

🖥 Результат:

 47.804
    47.804

7.3f означає: float з 3 десятковими знаками, відформатований на 7 символів (за замовчуванням — вирівняно праворуч). 10.3f робить те саме на 10 символах. Десятковий роздільник . (і знак - для від'ємних чисел) також враховується як окремий символ.

3.4.4 Інші деталі про f-рядки

Щоб вивести літеральні фігурні дужки у f-рядку, їх потрібно подвоїти, щоб екранувати від форматування:

print(f"Літеральні дужки {{}} або {{ або }}"
      f"а для форматування {10}")

🖥 Результат:

Літеральні дужки {} або { або }а для форматування 10

Якщо не помістити змінну для форматування між фігурними дужками, це призведе до помилки:

print(f"дужки без змінної {}")

🖥 Результат:

SyntaxError: f-string: valid expression required before '}'

f-рядок є незалежним від функції print(). Якщо передати f-рядок функції print(), Python спочатку обчислить f-рядок, і саме отриманий рядок буде виведено на екран — так само, як в інструкції print(5 * 5) спочатку обчислюється множення, а потім результат виводиться:

vidsotok_pravylnykh = ((4500 + 2575) / 14800) * 100
print(f"{vidsotok_pravylnykh:10.3f}")
print(type(f"{vidsotok_pravylnykh:10.3f}"))

🖥 Результат:

    47.804
<class 'str'>

Python розглядає результат f-рядка як рядок, і функція type() це підтверджує.

3.4.5 Вирази у f-рядках

Надзвичайно потужна функція f-рядків — підтримка виразів Python усередині фігурних дужок: туди можна безпосередньо помістити операцію або виклик функції:

print(f"Результат 5 * 5 дорівнює {5 * 5}")
print(f"Результат операції з floats: {(4.1 * 6.7)}")
print(f"Мінімум дорівнює {min(1, -2, 4)}")
zminna = 2
print(f"Тип {zminna} — {type(zminna)}")

🖥 Результат:

Результат 5 * 5 дорівнює 25
Результат операції з floats: 27.47
Мінімум дорівнює -2
Тип 2 — <class 'int'>

Можливості f-рядків численні; для повного огляду варіантів форматування зверніться до офіційної документації Python з форматованого виведення¹ (англійською).

3.5 Науковий запис

Для дуже великих або дуже малих чисел форматоване виведення дозволяє показати число в науковому записі (у вигляді степеня 10) за допомогою літери e:

print(f"{1_000_000_000:e}")
print(f"{0.000_000_001:e}")

🖥 Результат:

1.000000e+09
1.000000e-09

Можна також визначити кількість цифр після коми:

avogadro_number = 6.022_140_76e23
print(f"{avogadro_number:.0e}")
print(f"{avogadro_number:.3e}")
print(f"{avogadro_number:.6e}")

🖥 Результат:

6e+23
6.022e+23
6.022141e+23
