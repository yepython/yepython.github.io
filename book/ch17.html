РОЗДІЛ 17. Малюємо з SimpleGraphics

SimpleGraphics — це спеціалізована графічна бібліотека для мови програмування Python, створена для спрощення роботи з двовимірною (2D) комп'ютерною графікою. Її створив Бен Стефенсон (Ben Stephenson), викладач інформатики з Університеті Калгарі (University of Calgary, Канада), як навчальний інструмент для студентів перших курсів та початківців, які лише знайомляться з програмуванням.

Пакет працює поверх стандартної графічної бібліотеки Python під назвою tkinter. Головна мета створення SimpleGraphics — усунути складність синтаксису професійних графічних інструментів: вона дозволяє малювати фігури, керувати кольорами та обробляти прості події миші й клавіатури за допомогою всього кількох рядків зрозумілого коду.

Для тих, хто тільки починає свій шлях у програмуванні, ця бібліотека має низку ключових переваг:

  • Мінімальний поріг входження. На відміну від стандартних бібліотек, вам не потрібно розуміти концепції об'єктно-орієнтованого програмування (ООП), створення екземплярів класів (Canvas), «пакування» віджетів чи роботу з контекстом малювання. Ви просто викликаєте функції: line(), rect(), setOutline().
  • Відсутність громіздкого шаблонного коду. Щоб відкрити вікно та намалювати коло у звичайних бібліотеках, часто потрібно написати 10–15 рядків налаштувань. У SimpleGraphics достатньо двох рядків, щоб почати малювати:
from SimpleGraphics import *
circle(200, 200, 150)

  • Миттєвий візуальний відгук. Початківцям важко довгий час працювати лише з текстовою консоллю. SimpleGraphics дозволяє одразу бачити результат своєї логіки (наприклад, роботу циклів чи умовних операторів) у вигляді візуальних паттернів, анімацій або простих ігор.
  • Прихована автоматизація. Бібліотека самостійно керує оновленням екрана (screen refresh) та утримує вікно відкритим, поки користувач його не закриє, що звільняє від необхідності вивчати складні системні цикли подій (event loops) на ранніх етапах.

Вивчення графіки за допомогою SimpleGraphics закладає фундаментальні концепції, які використовуються у дорослих та професійних інструментах (таких як Pygame, Tkinter, Arcade, PyQt, Kivy або навіть HTML5 Canvas у веброзробці). Вона слугує ідеальною запорукою для подальшого навчання:

Математична система координат (де центр у точці (0, 0), а вісь Y йде вгору) у комп'ютерній графіці майже не використовується. SimpleGraphics привчає розробника до екранної системи координат, де (0, 0) — це лівий верхній кут, а Y росте вниз. Цей принцип абсолютно ідентичний у Pygame, веброзробці (CSS/Canvas) та UI-фреймворках. Користувач, який зрозумів це в SimpleGraphics, не матиме просторової дезорієнтації в майбутньому.

Вказівки керування кольором тла (background()), контуру (setOutline()) та заливки (setFill()) формують чітке розуміння того, як працює графічний конвеєр: спочатку ми обираємо «пензель», а вже потім малюємо фігуру. Цей «режим малювання» є базовим для низькорівневої графіки.

SimpleGraphics має прості функції для організації анімації та зчитування натискань клавіш і позиції миші. Проектуючи анімацію (наприклад, коло, що рухається по екрану завдяки зміні координати X у циклі while), користувач вивчає фундаментальний алгоритм комп'ютерної графіки:

  1. Очистити екран.
  2. Перерахувати нові координати об'єктів.
  3. Намалювати об'єкти заново.

Коли у майбутньому розробник перейде до Pygame чи Arcade, він побачить там абсолютно ті самі механізми (Game Loop, Event Handling, Blitting), але вже реалізовані за допомогою об'єктно-орієнтованого підходу. Перехід буде плавним, оскільки сама математична та алгоритмічна основа вже відпрацьована в SimpleGraphics.

17.1. Встановлення бібліотеки SimpleGraphics

Якщо ви використовуєте онлайнове середовище програмування ЄPython, тоді встановлювати нічого не потрібно — вона є складовою частиною цього середовища. У більшості інших середовищ бібліотеку потрібно встановити. SimpleGraphics доступна у глобальному сховищі PyPI під назвою simplegraphics-python, тому процес її встановлення зводиться до стандартного використання пакетного менеджера pip. Проте, залежно від операційної системи та обраного середовища розробки (IDE), цей процес має свої нюанси.

Thonny IDE

Thonny — це чудове середовище для початківців, яке має власний ізольований Python та графічний менеджер пакетів. Це найпростіший спосіб.

  1. Відкрийте Thonny.
  2. У верхньому меню оберіть: Інструменти (Tools) → Керування пакетами (Manage packages).
  3. У полі пошуку введіть: simplegraphics-python.
  4. Натисніть кнопку Знайти на PyPI (Search on PyPI).
  5. Клацніть на знайдений пакет у списку та натисніть кнопку Встановити (Install).
  6. Дочекайтеся завершення встановлення. Готово!
IDLE (стандартне середовище Python)

IDLE використовує ту версію Python, яка встановлена безпосередньо у вашій операційній системі. Для встановлення бібліотеки потрібно скористатися терміналом/командним рядком вашої ОС.

  1. Закрийте IDLE (бажано).
  2. Відкрийте термінал вашої ОС (Командний рядок у Windows, Термінал у macOS/Linux).
  3. Введіть команду встановлення (команди для конкретних ОС наведено у п. 17.2.2).
  4. Після успішного встановлення запустіть IDLE — бібліотека буде доступна.
PyCharm

PyCharm за замовчуванням створює окреме віртуальне середовище (venv) для кожного нового проєкту. Пакет потрібно встановлювати безпосередньо в проєкт.

Спосіб через графічний інтерфейс:

  1. Відкрийте ваш проєкт у PyCharm.
  2. Перейдіть до Settings (Windows: File → Settings, macOS: PyCharm → Settings).
  3. Оберіть Project: [Назва проєкту] → Python Interpreter.
  4. Натисніть на значок + (або кнопку Install Sign у новіших версіях).
  5. У рядку пошуку введіть simplegraphics-python.
  6. Виберіть пакет та натисніть Install Package внизу вікна.

Спосіб через вбудований термінал:

  1. Внизу екрана PyCharm відкрийте вкладку Terminal.
  2. Введіть команду pip install simplegraphics-python та натисніть Enter.
Visual Studio Code (VS Code)

VS Code працює через термінал і вимагає, щоб ви переконалися, що пакет встановлюється саме в той інтерпретатор, який зараз обрано в редакторі.

  1. Відкрийте папку з вашим проєктом у VS Code.
  2. Переконайтеся, що інтерпретатор Python обрано (знизу праворуч або через комбінацію Ctrl+Shift+P / Cmd+Shift+P → Python: Select Interpreter).
  3. Відкрийте вбудований термінал (Ctrl + ~ або Terminal → New Terminal).
  4. Якщо ви використовуєте віртуальне середовище, перевірте, чи воно активоване (на початку рядка термінала має бути щось на кшталт (.venv)).
  5. Запустіть команду встановлення:
    • Windows: pip install simplegraphics-python
    • macOS/Linux: pip3 install simplegraphics-python

Особливості встановлення в різних ОС

Якщо ви встановлюєте бібліотеку через глобальний термінал (для IDLE або VS Code без venv), команди відрізняються залежно від платформи.

Windows

  1. Натисніть клавіші Win + R, введіть cmd та натисніть Enter (відкриється Командний рядок).

  2. Введіть таку команду:

    py -m pip install simplegraphics-python

    (Використання утиліти py гарантує, що пакет встановиться для правильної версії Python, якщо їх у системі кілька.)

macOS

На комп'ютерах Mac стандартна команда python може викликати застарілі системні версії або видавати помилку, тому використовується python3.

  1. Відкрийте програму Термінал (Terminal) (через Spotlight або папку Програми → Утиліти).

  2. Виконайте команду:

    python3 -m pip install simplegraphics-python

Важливо для macOS: оскільки бібліотека побудована на базі графічного рушія Tkinter, на деяких версіях macOS (особливо з чипами Apple Silicon M1/M2/M3) може виникнути помилка ModuleNotFoundError: No module named '_tkinter'. Якщо це сталося, потрібно встановити підтримку Tcl/Tk через менеджер пакетів Homebrew, виконавши в Терміналі: brew install python-tk.

Linux (Ubuntu / Debian / Fedora)

У деяких дистрибутивах Linux пакетний менеджер pip та модуль tkinter відключені від Python за замовчуванням з міркувань безпеки. Їх потрібно встановити через системний менеджер пакетів.

  1. Відкрийте Термінал.

  2. Оновіть репозиторії та встановіть pip разом із tkinter (приклад для Ubuntu/Debian):

    sudo apt update
sudo apt install python3-pip python3-tk

  3. Після цього встановіть саму бібліотеку графіки:

    python3 -m pip install simplegraphics-python

Примітка: якщо система Linux видає помилку про «externally-managed-environment», рекомендується встановлювати бібліотеку всередині віртуального середовища (venv), в Thonny IDE або скориставшись командою python3 -m pip install simplegraphics-python --break-system-packages.

Як перевірити, чи все працює?

Після встановлення створіть новий файл Python (наприклад, test_graphics.py) та запустіть наступний мінімальний код:

from SimpleGraphics import *

# Малюємо синє коло по центру
background("white")
setFill("blue")
circle(400, 300, 100)

# Малюємо лінію
setOutline("red")
line(0, 0, 400, 300)

Якщо перед вами з'явилося вікно з синім колом та червоною лінією — вітаємо, бібліотека встановлена правильно і готова до роботи!

17.2. Система координат

Система координат у графічній бібліотеці SimpleGraphics базується на стандартній екранній (комп'ютерній) системі координат. Вона суттєво відрізняється від традиційної декартової системи координат, яку вивчають у шкільному курсі математики.

Головні принципи екранної координатної системи

  • Початок координат (0, 0) розташований у верхньому лівому кутку вікна програми.
  • Вісь X (горизонтальна вісь): спрямована зліва направо. Значення X зростають, коли ви рухаєтеся вправо, що збігається з математичною системою.
  • Вісь Y (вертикальна вісь): спрямована згори вниз. Це головна відмінність від математики: чим більшим є значення Y, тим нижче на екрані буде розташована точка.

Приклад: точка з координатами (0, 0) — це найверхніший лівий піксель вікна. Точка (100, 200) означає, що об'єкт зміщено на 100 пікселів вправо та на 200 пікселів вниз від верхнього лівого кутка.

Одиниці виміру та межі

  • Пікселі. Усі координати, відстані, ширина та висота у SimpleGraphics вимірюються в пікселях (точках екрана). На відміну від математичних графіків, тут немає абстрактних одиниць — кожен крок дорівнює одному фізичному пікселю вікна.
  • Розмір вікна за замовчуванням. При імпорті бібліотеки автоматично створюється вікно розміром 800 × 600 пікселів; за потреби цей розмір можна змінити функцією resize(width, height).
  • Якщо розмір вікна становить, наприклад, 600 × 400, то правий нижній куток матиме координати (599, 399), оскільки відлік починається з нуля.
  • Від'ємні координати та координати за межами вікна. Ви можете використовувати від'ємні числа (наприклад, x = -50) або числа, що перевищують розмір вікна. Бібліотека не видасть помилки, але ці точки просто опиняться за межами видимої області екрана (за лівою/верхньою або правою/нижньою межами відповідно).

Як координати застосовуються до фігур

Різні геометричні фігури використовують координати по-різному.

А. Лінії (line). Для малювання лінії потрібні дві пари координат — початкова точка (x1, y1) та кінцева точка (x2, y2):

line(50, 50, 200, 150)  # Проводить лінію з точки (50, 50) до точки (200, 150)

Б. Прямокутники (rect) та еліпси (ellipse). Для цих фігур початкові координати (x, y) задають верхній лівий кут уявної коробки (габаритного контейнера), в яку вписується фігура. Далі вказуються її ширина (width) та висота (height):

rect(100, 100, 200, 50)  # Малює прямокутник, верхній лівий кут якого в (100, 100)

Оскільки вісь Y напрямлена вниз, прямокутник буде рости від точки (100, 100) вниз та вправо. Нижній правий кут цього прямокутника матиме координати (100 + 200, 100 + 50) = (300, 150).

В. Текст (text). При виведенні тексту координати (x, y) за замовчуванням вказують на позицію розміщення тексту, проте SimpleGraphics дозволяє налаштовувати «якір» (anchor) відносно сторін світу ("n", "s", "e", "w"), щоб чітко керувати тим, де саме текст опиниться відносно заданої координатної точки.

Чому система саме така?

Така логіка (нуль у верхньому лівому кутку) є історичним стандартом для більшості графічних систем (включаючи tkinter, на базі якого побудована SimpleGraphics, а також HTML Canvas, Java Swing тощо). Це пов'язано з тим, як старі ЕЛТ-монітори формували зображення — електронний промінь сканував екран рядок за рядком, починаючи саме з лівого верхнього кутка і рухаючись вниз.

17.3 Малюємо, використовуючи SimpleGraphics

Після імпорту бібліотеки за допомогою команди from SimpleGraphics import * автоматично створюється графічне вікно розміром 800 × 600 пікселів.

Перший крок: малювання кола

Малювання круглих фігур виконується за допомогою функції circle().

from SimpleGraphics import *

# Малюємо коло
circle(400, 300, 100)

Пояснення аргументів circle(x, y, diameter):

  • x (перший аргумент, тут 400) — координата по горизонталі для лівого краю уявного квадрата, в який вписується коло.
  • y (другий аргумент, тут 300) — координата по вертикалі для верхнього краю уявного квадрата.
  • diameter (третій аргумент, тут 100) — діаметр кола в пікселях.

Керування кольором та лініями

Перш ніж переходити до складніших фігур, важливо навчитися фарбувати їх та змінювати товщину контуру. За замовчуванням фігури малюються чорним контуром без заливки. Змінити це можна за допомогою таких вказівок:

  • background("колір") — повністю перефарбовує тло всього вікна.
  • setOutline("колір") — встановлює колір ліній та контурів для усіх фігур, які будуть малюватися після цієї команди.
  • setFill("колір") — встановлює колір заливки (внутрішньої частини) для всіх наступних фігур.
  • setWidth(пікселі) — змінює товщину ліній контуру.
  • setColor("колір") — швидкий спосіб встановити водночас і колір контуру, і колір заливки однією командою (якщо не потрібно, щоб контур і заливка відрізнялися кольором).
from SimpleGraphics import *

background("lightgray")  # Робимо тло вікна сірим
setWidth(3)               # Збільшуємо товщину ліній контуру до 3 пікселів
setOutline("red")         # Наступні фігури матимуть червону межу
setFill("blue")           # Наступні фігури будуть синіми всередині

# Тепер коло буде синім із товстим червоним контуром на сірому тлі
circle(200, 200, 80)

Малювання прямокутників та інших геометричних фігур

Тепер, коли ми вміємо налаштовувати кольори, розглянемо інші базові вказівки побудови в логічному порядку — від простих ліній до складних багатокутників.

Прямокутник — rect(x, y, width, height). Параметри x та y вказують на розташування верхнього лівого кута прямокутника, а width та height задають його ширину й висоту.

from SimpleGraphics import *

setFill("yellow")
# Малюємо жовтий прямокутник
rect(50, 50, 200, 100)  # Початок у (50, 50), ширина 200, висота 100

Лінія — line(x1, y1, x2, y2). Проводить пряму лінію між двома точками на екрані. (x1, y1) — це координати початку лінії, а (x2, y2) — координати її кінця. Якщо більше, ніж одна пара координат — будується ламана:

from SimpleGraphics import *

# Проводимо ламану
line(10, 10, 100, 100, 200, 10, 300, 100, 400, 10)

Еліпс — ellipse(x, y, width, height). Параметри x та y вказують на розташування верхнього лівого кута уявного прямокутника, що містить це еліпс, а width та height задають його ширину й висоту.

from SimpleGraphics import *

setFill("yellow")
# Малюємо жовтий еліпс(овал)
ellipse(50, 50, 200, 100)  # Початок у (50, 50), ширина 200, висота 100

setFill("pink")
# Малюємо рожевий еліпс
ellipse(350, 50, 100, 200)  # витягнутий вертикально

Дуга та сектор — arc() і pieSlice(). Вони приймають параметри (x, y, width, height, start_angle, extent). Спочатку визначається уявний прямокутник (як для еліпса), а потім вирізається потрібна частина за кутами. start_angle — початковий кут у градусах, extent — величина(розмах) дуги.

  • arc — малює лише дугу (частину кола).
  • pieSlice — малює заповнений сектор круга.
from SimpleGraphics import *
background("lightgray")
setFill("lightgray")
# Дуга з початковим кутом 45 градусів та розмахом 90 градусів
arc(200, 300, 200, 200, 45, 90)
setFill("blue")
# Сектор із такими ж параметрами (буде зафарбований синім кольором)
pieSlice(350, 300, 200, 200, 45, 90)

Багатокутник — polygon(x1, y1, x2, y2, x3, y3, ...). Дозволяє будувати фігури з будь-якою кількістю кутів (трикутники, зірки, ромби). Ви просто по черзі перераховуєте пари координат усіх вершин, які будуть автоматично з'єднані між собою, а остання - точка з першою.

from SimpleGraphics import *

setFill("green")
# Передаємо 3 пари координат, щоб отримати зелений трикутник
polygon(10, 10, 50, 250, 200, 100)

Плавні криві — curve() та blob(). Ці функції створюють заокруглені, хвилясті лінії через задані точки.

  • curve — малює відкриту плавну лінію.
  • blob — автоматично замикає початкову і кінцеву точки кривої та заповнює фігуру зсередини.
from SimpleGraphics import *

# Малювання звичайної кривої
curve(100, 100, 150, 150, 200, 100)

# Малювання заповненої кривої
blob(200, 200, 300, 400, 400, 200)

Текст — text(x, y, "текст", align="center"). Виводить напис на екран у вказаних координатах (x, y). За допомогою параметра align можна вказати вирівнювання (наприклад, по центру).

from SimpleGraphics import *

text(400, 300, "Привіт, SimpleGraphics!", align="center")

Більш складніші малюнки

Щоб створити повноцінний малюнок, ми поєднуємо примітиви.

Важливе правило: фігури, які записані в коді нижче, будуть малюватися поверх тих, що записані вище.

Зразок 1. Побудова «Будиночка». Малюнок складається з прямокутної стіни, трикутного даху (через polygon), прямокутних дверей та круглого віконця.

from SimpleGraphics import *
# Вікно
resize(700, 500)
background("skyblue")

# Земля
setColor("green")
rect(0, 280, 800, 260)

# Сонце
setColor("yellow")
circle(50, 50, 60)

# Будинок
setColor("gold")
rect(130, 190, 350, 180)

# Дах
setFill("brown")
polygon(130, 190,305, 80,480, 190)

# Двері
setColor("chocolate")
rect(260, 230, 90, 140)

setWidth(4)
# Ліве вікно
setColor("white")
rect(150, 230, 90, 90)
setFill("aqua")
rect(150, 230, 90, 90)
setColor("white")
line(195, 230, 195, 320)
line(150, 275, 240, 275)

# Праве вікно
setColor("white")
rect(370, 230, 90, 90)
setFill("aqua")
rect(370, 230, 90, 90)
setColor("white")
line(415, 230, 415, 320)
line(370, 275, 460, 275)

# Дерево - стовбур
setColor("brown")
rect(600, 180, 25, 190)
# Крона
setColor("olivedrab")
ellipse(550, 20, 120, 240)

Зразок 2. Побудова «Кораблика». Тут ми використовуємо багатокутник (polygon) для створення форми трапеції (корпусу човна), лінію для щогли та заповнену криву (blob) для створення ефекту хвиль моря.

from SimpleGraphics import *

background("skyblue")

# Корпус корабля
setColor("red")
polygon(60, 260,
        340, 260,
        300, 300,
        100, 300)

# Щогла
setColor("black")
setWidth(3)
line(180, 40, 180, 260)

# Велике вітрило
setColor("cyan")
polygon(180, 80,
        180, 240,
        300, 240)

# Маленьке вітрило
setColor("blue")
polygon(180, 120,
        180, 240,
        90, 240)

# Прапорець
setColor("yellow")
rect(180, 45, 35, 10)

# Море
setColor("blue")
rect(0, 300, 800, 300)

17.4. Додаткові можливості

Розглянемо ще кілька корисних прикладів, які доповнюють базовий набір функцій SimpleGraphics: стилізовані лінії, ламані та згладжені криві, складні фігури за списком координат, прямокутники, що перекриваються, еліпс і багатокутник-зірку.

Лінії різних стилів

Три лінії, намальовані в різних стилях. Побачимо, як змінювати колір та товщину за допомогою setColor() та setWidth().

from SimpleGraphics import *
# Встановлюємо зелений фон для всього вікна
background("springgreen")

# Лінія 1
setWidth(3)
setColor("black")
line(20, 20, 180, 20)

# Лінія 2
setWidth(5)
setColor("red")
line(280, 20, 480, 270)

# Лінія 3 (Товста лінія)
setWidth(15)
setColor("darkblue")
line(150, 170, 150, 370)

Ламана та згладжена крива

Лінії не обов'язково мають бути прямими. Більш загальний тип лінії може складатися з багатьох прямих сегментів — а якщо замінити line() на curve(), ті самі точки утворять плавний, згладжений вигин.

from SimpleGraphics import *

setWidth(3)
setColor("violet")

# Різкий згин (передаємо більше двох точок)
line(50, 10, 20, 80, 250, 160)

setColor("green")
# Кривий згин (функція curve() будує згладжену криву)
curve(50, 10, 20, 80, 250, 160)

Складна крива за списком координат: виноградна лоза

Завдання полягає в тому, щоб намалювати складну фігуру, використовуючи списки координат, попередньо виміряні, наприклад, з ескізу на паперу.

from SimpleGraphics import *

# Координати, виміряні з ескізу на папері
vine_x = [23, 20, 11, 9, 29, 52, 56, 39, 24, 32, 53, 69, 63, 47, 35, 35, 51, 82, 116, 130, 95, 67, 95, 114, 95, 78, 95, 103, 95, 85, 95, 94]
vine_y = [36, 44, 39, 22, 16, 32, 56, 72, 91, 117, 125, 138, 150, 151, 140, 123, 107, 92, 70, 41, 5, 41, 66, 41, 24, 41, 53, 41, 33, 41, 41, 39]

# Об'єднуємо списки X та Y в один черговий список
pts = []
for i in range(len(vine_x)):
    pts.append(vine_x[i])
    pts.append(vine_y[i])

# Малюємо виноградну лозу як згладжену криву
curve(*pts)

Перекривання фігур

Якщо намалювати кілька прямокутників зі зсувом один за одним, кожен наступний буде перекривати попередній.

from SimpleGraphics import *

# Темно-синій прямокутник
setColor("darkblue")
rect(10, 30, 70, 90)

# Темно-червоний прямокутник (зміщений, малюється другим)
setColor("darkred")
rect(30, 50, 70, 90)

# Темно-зелений прямокутник (малюється останнім, тому "зверху")
setColor("darkgreen")
rect(50, 70, 70, 90)

Коло, овал та еліпс

Найкращий спосіб намалювати коло або еліпс — використати методи ellipse(x, y, w, h) або circle(x, y, d). На відміну від кола, еліпс дозволяє окремо вказати ширину та висоту габаритного прямокутника.

# kolo.py
from SimpleGraphics import *

setColor("cyan")
# ellipse малює еліпс, вписаний у прямокутник (x, y, ширина, висота)
ellipse(20, 20, 100, 100)

# circle малює коло за центром та діаметром
# circle(70, 70, 100)

17.5 Робота з текстом

Текст може бути різноманітним, тож нам потрібно мати можливість керувати обраними шрифтами, розміром, кольором та розміщенням. SimpleGraphics спрощує це, надаючи функції text() та setFont().

from SimpleGraphics import *

# За замовчуванням текст малюється чорним кольором
text(150, 20, "Розмір тексту за замовчуванням виглядає як 10 пунктів")

Тип, розмір і колір шрифту

Колір та стиль тексту керуються відповідними функціями setColor() та setFont(). Функція setFont() приймає назву шрифту, розмір у пунктах і, за потреби, стиль ("bold", "italic").

from SimpleGraphics import *

setColor("red")
setFont("SansSerif", 20)
text(200, 20, "Звичайний текст SansSerif 20")

setColor("blue")
setFont("Arial", 20)
text(200, 50, "Звичайний текст Arial 20")

setColor("green")
setFont("Courier", 20, "bold")
text(200, 80, "Напівжирний текст Courier 20")

setColor("purple")
setFont("BookAntiqua", 20, "italic")
text(200, 110, "Курсив BookAntiqua 20")

Розміщення тексту

Розмістити текст точно там, де ви хочете, може бути складно через висоту шрифту та міжсимвольний інтервал. Функція text(x, y, what) за замовчуванням центрує текст (align="c") відносно точки (x, y). Ви можете змінити вирівнювання, передавши параметр align="w" (ліворуч), align="e" (праворуч) тощо.

from SimpleGraphics import *

# Довгий текст
jolly_text = "Книги – морська глибина: Хто в них пірне аж до дна,той, хоч і труду мав досить, дивнії перли виносить"

setColor("blue")
setFont("Arial", 12)

# Вирівнювання по лівому краю відносно точки (120, 40)
text(220, 40, jolly_text, align="w")

Перегляд доступних шрифтів

Дізнаємося, які шрифти доступні, та виведемо їхній зразок, перебираючи список у циклі.

from SimpleGraphics import *

# Список поширених шрифтів
shryfty = ["Arial", "Courier", "Times New Roman", "Verdana", "Georgia"]
zrazok = ": abcdefghij_HIJK_12340"

y_pozyciia = 30
setColor("black")

for shryft in shryfty:
    setFont(shryft, 24)
    # Вирівнюємо по лівому краю, щоб текст йшов рівними стовпчиками
    text(50, y_pozyciia, f"{shryft} {zrazok}", align="w")
    y_pozyciia += 50

  1. Зоряне небо. Використовуючи цикл for та функцію random (модуль random), намалюйте на темному тлі 50 маленьких «зірок» у випадкових місцях вікна.
import random
from SimpleGraphics import *

# Створюємо функцію для малювання однієї зірки

    setFill("white")
    circle(x, y, size)

# --- Основна програма ---
background("darkblue")

# Цикл для малювання 150 випадкових зірок
for i in range(150):
    # Генеруємо випадкові координати у межах нашого вікна
    random_x = random.randint(0, 800)
    random_y = random.randint(0, 600)
    
    # Генеруємо випадковий розмір (діаметр від 2 до 6 пікселів)
    random_size = random.randint(2, 6)
    
    # Викликаємо нашу функцію для малювання зірки
    star(random_x, random_y, random_size)

    • Посилання: https://cspages.ucalgary.ca/~bdstephe/SG_tutorial.pdf (або шукайте на сторінках курсів автора).

  2. Tutorial 2: Using Images with the SimpleGraphics Library (image_tutorial.pdf)

    • Робота з зображеннями (GIF, PPM, PGM): завантаження, відображення, маніпуляція пікселями, створення нових зображень, ефекти (grayscale, mirror тощо).
    • Посилання: https://cspages.ucalgary.ca/~bdstephe/217_W14/image_tutorial.pdf.
